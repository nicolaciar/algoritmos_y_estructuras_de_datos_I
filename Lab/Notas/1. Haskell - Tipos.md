# 🧠 **1. ¿Qué es un tipo en Haskell?**

Un **tipo** es una _clasificación estática_ de valores. Cada expresión en Haskell pertenece a un tipo, que determina:

- **qué valores puede tomar**
    
- **qué operaciones puedes hacer con él**
    
- **cómo se interpreta en memoria o en el lenguaje**
    

Como en matemáticas, el tipo describe el “dominio” de los valores con los que trabaja la función.

Por ejemplo:

```haskell
5 :: Int
"hola" :: String
True :: Bool
```

En Haskell, **los tipos son estáticos y se verifican antes de ejecutar el programa**: ningún programa bien tipado puede fallar en tiempo de ejecución por problemas de tipos (al menos no por eso).

---

# 🎯 **2. Tipado fuerte y estático**

Haskell tiene dos propiedades clave:

### ✔ Tipado estático

El tipo de toda expresión debe ser conocido **en tiempo de compilación**.

No puede ocurrir:

```haskell
x = 5
x = "hola"   -- ❌ imposible
```

### ✔ Tipado fuerte

Nunca puedes “forzar” una operación entre tipos incompatibles:

```haskell
5 + "hola"  -- ❌ jamás compila
```

Esto es muy distinto de lenguajes como Python o JS, que pueden mezclar cosas en tiempo de ejecución.

# 🧱 **3. Tipos básicos (primitivos)**

Los más elementales, provistos por la biblioteca estándar:

|Tipo|Descripción|
|---|---|
|`Int`|Entero de tamaño fijo (palabra de máquina)|
|`Integer`|Entero arbitrario, sin límite teórico|
|`Float`, `Double`|Reales de punto flotante|
|`Bool`|`True` / `False`|
|`Char`|Un carácter|
|`String`|Alias de `[Char]`|

Ejemplos:

```haskell
42       :: Int
1234567  :: Integer
3.14     :: Double
'a'      :: Char
"hola"   :: String
```

# 📦 **4. Tipos compuestos (estructurados)**

Estos se construyen a partir de otros.

## **a) Tuplas**

```haskell
(3, "a")        :: (Int, String)
(True, 2.5, 'c') :: (Bool, Double, Char)
```

Son **heterogéneas y de tamaño fijo**.

## **b) Listas**

```haskell
[1, 2, 3] :: [Int]
"hola"   :: [Char]
```

Las listas son **homogéneas y de tamaño dinámico**, pero responden a una misma estructura recursiva:

```haskell
[]                 -- lista vacía
x : xs             -- un elemento seguido de otra lista
```

# ✨ **5. Tipos algebraicos (ADTs)**

Haskell permite **definir nuevos tipos**, combinándolos por suma o producto.

Ejemplo: producto (struct-like)

```haskell
data Punto = P Float Float
```

Ejemplo: suma (enum-like)

```haskell
data Color = Rojo | Verde | Azul
```

Ejemplo sum + product:

```haskell
data Figura
    = Circulo Float
    | Rectangulo Float Float
```

Estos son **la esencia del diseño de datos en Haskell**: permiten representar estructuras ricas con semántica clara y verificación estática.

---

# 🧩 **6. Sinónimos de tipos (type synonyms)**

No generan nuevos valores, solo **nombres alternativos**:

```haskell
type Nombre = String
type Edad = Int
```

Son útiles para documentación y claridad.

---

# 🎚️ **7. Polimorfismo paramétrico**

Una función puede ser **general para cualquier tipo**, sin depender de su contenido.

Ejemplo clásico:

```haskell
id :: a -> a
id x = x
```

`a` es un **parámetro de tipo**: no sabemos qué es, pero no importa para la función.

Otro ejemplo:

```haskell
length :: [a] -> Int
```

`length` no necesita saber el contenido de la lista.

👉 Este polimorfismo es **puro**, no inspecciona valores, solo estructura.

---

# 🏷️ **8. Polimorfismo por clases de tipos (typeclasses)**

Permite **sobrecargar funciones** según qué operaciones sean válidas para un tipo.

Ejemplos:

```haskell
(+) :: Num a => a -> a -> a
```

La restricción `Num a =>` indica:

> “esto funciona para cualquier tipo `a` que sea instancia de la clase `Num`”

Esto no es polimorfismo paramétrico puro, sino **adyacente a ad-hoc**.

Las clases de tipos fundamentales:

- `Eq` (comparación)
    
- `Ord` (orden total)
    
- `Show` (conversion a string)
    
- `Read` (parseo desde string)
    
- `Num`, `Fractional` (operaciones numéricas)
    

Los ADTs pueden ser instancias:

```haskell
data Color = Rojo | Verde | Azul deriving (Show, Eq)
```

# 🚦 **9. Naturaleza de los tipos en Haskell**

Consideraciones profundas:

### ✔ Los tipos describen el comportamiento semántico, no la representación física

Ejemplo: `Bool` es matemáticamente:

```haskell
data Bool = True | False
```

No importa cómo se represente en bits.

### ✔ Los tipos son parte del lenguaje

Esto no es como en C o Java donde el “tipo” es más incidental. En Haskell:

- los tipos forman su propia sintaxis
    
- hay **funciones entre tipos**
    
- el compilador los prueba como si fueran teoremas
    

### ✔ El sistema de tipos es rico

No solo verifica compatibilidad: **infieren, abstraen y garantizan propiedades semánticas**.

# 📏 **10. Consideraciones importantes de diseño**

Como buen  matemático :

- **Evita usar `String` cuando puedas usar nuevos tipos**  
    Más semántica, menos errores.
    
- **Prefiere Integer a Int si necesitas seguridad numérica**
    
- **Diseña tus propios ADTs** en lugar de estructuras sueltas
    
- **Aprovecha typeclasses** para evitar duplicación
    
- **Piensa los tipos como invariantes**  
    Cada tipo expresa condiciones semánticas del problema.
    
- **Especificar tipos es parte del contrato**  
    No solo “documenta”: **el compilador prueba su validez**.

# ⭐ **11. Cómo leer firmas de tipo**

Si ves:

```haskell
f :: Num a => a -> a -> a
```

Esto significa:

> Para cualquier `a` que tenga operaciones numéricas válidas, `f` toma dos valores de tipo `a` y devuelve otro de tipo `a`.

Si ves:

```haskell
map :: (a -> b) -> [a] -> [b]
```

Esto significa:

> Dada una función que transforma `a` en `b`, produce una lista transformada.
